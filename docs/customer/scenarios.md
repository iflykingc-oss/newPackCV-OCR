# 🎯 场景 Schema 详解

> 8 大行业场景的字段定义、必填项、最佳实践。

## 场景总览

| 场景 | scenario | 必填字段 | 可选字段 |
|------|----------|---------|---------|
| 📦 包装 | `packaging` | 品牌/品名 | 规格/生产日期/保质期/厂家/条形码/成分 |
| 💰 金融票据 | `finance_receipt` | 票据类型/金额/日期 | 商户/收付款方/税号/币种/项目明细 |
| 🏦 银行流水 | `finance_statement` | 账户/币种 | 交易明细/余额/起止日期/银行名 |
| 💊 医药 | `pharmaceutical` | 通用名/批准文号 | 规格/生产企业/有效期/成分/适应症 |
| 📋 合同 | `contract` | 合同编号/甲乙方/金额 | 有效期/签署日期/条款摘要 |
| 🪪 证件 | `id_card` | 证号/姓名 | 性别/出生日期/住址/有效期/签发机关 |
| 🚚 物流 | `logistics` | 运单号/收件人/寄件人 | 品名/重量/运费/签收状态 |
| 📄 通用文档 | `general_document` | 标题 | 摘要/关键词/正文 |

## 📦 packaging (产品包装)

**适用文档**: 产品包装盒、标签、说明书外包装

**必填字段**:
- `品牌` (brand) - 制造商品牌
- `品名` (product_name) - 产品名称

**推荐字段**:
- `规格` (specification) - 如 500ml/250g
- `生产日期` (production_date) - ISO 8601
- `保质期` (expiry_date) - ISO 8601 或天数
- `厂家` (manufacturer) - 生产企业
- `条形码` (barcode) - 数组,每个条码一个对象
- `成分` (ingredients) - 成分列表

**特殊能力**:
- ✅ 条形码/二维码自动识别
- ✅ 弯曲文字校正 (TPS)
- ✅ 多语言 (109 种)
- ✅ 低光照增强 (CLAHE)

**示例响应**:
```json
{
  "scenario": "packaging",
  "fields": {
    "品牌": "可口可乐",
    "品名": "可口可乐 经典原味",
    "规格": "500ml",
    "生产日期": "2024-01-10",
    "保质期": "2025-01-10",
    "厂家": "可口可乐饮料有限公司",
    "条形码": [{"type": "EAN-13", "value": "6901234567890"}]
  }
}
```

## 💰 finance_receipt (金融票据)

**适用文档**: 增值税发票、电子发票、收据、回单

**必填字段**:
- `票据类型` (receipt_type) - 增值税普通发票/电子发票/收据等
- `金额` (amount) - 数字字符串,带币种符号
- `日期` (date) - ISO 8601

**推荐字段**:
- `商户` (merchant) - 销售方
- `买方` (buyer) - 购买方
- `税号` (tax_id) - 统一社会信用代码
- `币种` (currency) - CNY/USD/EUR
- `项目明细` (line_items) - 数组,每项含 名称/数量/单价/小计

**特殊能力**:
- ✅ 表格结构化 (HTML/Markdown)
- ✅ 印章检测 + 真伪对比
- ✅ 多种币种识别

**示例响应**:
```json
{
  "scenario": "finance_receipt",
  "fields": {
    "票据类型": "增值税电子普通发票",
    "金额": "¥1,234.56",
    "日期": "2024-01-15",
    "商户": "ACME咖啡有限公司",
    "税号": "91110000123456789X",
    "项目明细": [
      {"名称": "美式咖啡", "数量": "2", "单价": "30.00", "小计": "60.00"},
      {"名称": "提拉米苏", "数量": "1", "单价": "48.00", "小计": "48.00"}
    ]
  }
}
```

## 🏦 finance_statement (银行流水)

**适用文档**: 银行对账单、月度账单

**必填字段**:
- `账户` (account) - 账号或卡号(脱敏)
- `币种` (currency) - CNY/USD/HKD

**推荐字段**:
- `银行` (bank_name) - 工商银行/招商银行等
- `账户名` (account_name) - 脱敏
- `起止日期` (period) - {from, to}
- `期初余额` (opening_balance)
- `期末余额` (closing_balance)
- `交易明细` (transactions) - 数组

**交易明细结构**:
```json
{
  "date": "2024-01-15",
  "type": "支出",        // 收入/支出
  "amount": "1,000.00",
  "balance": "10,234.56",
  "counterparty": "ACME公司",
  "description": "货款"
}
```

## 💊 pharmaceutical (医药)

**适用文档**: 药品包装、说明书、处方

**必填字段**:
- `通用名` (generic_name) - 化学名/通用名
- `批准文号` (approval_number) - 国药准字 H20012345

**推荐字段**:
- `商品名` (trade_name)
- `规格` (specification) - 0.25g*24片
- `生产企业` (manufacturer)
- `有效期` (expiry_date)
- `成分` (ingredients) - 化学成分列表
- `适应症` (indications) - 适应症/功能主治
- `用法用量` (dosage)
- `不良反应` (side_effects)

**特殊能力**:
- ✅ 高精度 OCR (109 语言含拉丁文)
- ✅ 表格结构化 (成分表)
- ✅ 印章检测 (国药准字印章)

**示例响应**:
```json
{
  "scenario": "pharmaceutical",
  "fields": {
    "通用名": "阿莫西林胶囊",
    "批准文号": "国药准字H20012345",
    "规格": "0.25g*24粒",
    "生产企业": "华北制药股份有限公司",
    "有效期": "2025-12-31",
    "成分": ["阿莫西林 0.25g"],
    "适应症": "适用于敏感菌所致的感染"
  }
}
```

## 📋 contract (合同)

**适用文档**: 劳动合同、商业合同、租赁合同

**必填字段**:
- `合同编号` (contract_id)
- `甲方` (party_a) - 委托方
- `乙方` (party_b) - 承接方
- `金额` (amount)

**推荐字段**:
- `合同类型` (contract_type) - 劳动合同/销售合同/服务合同
- `有效期` (validity_period) - {from, to}
- `签署日期` (signing_date)
- `主要条款` (key_clauses) - 数组
- `违约责任` (liability_terms)
- `争议解决` (dispute_resolution) - 仲裁/诉讼

**特殊能力**:
- ✅ 长文本理解 (LLM 摘要)
- ✅ 印章检测 (合同专用章/法人章)
- ✅ 多页文档

## 🪪 id_card (证件)

**适用文档**: 居民身份证、护照、驾照

**必填字段**:
- `证号` (id_number) - 身份证号/护照号
- `姓名` (name)

**推荐字段**:
- `性别` (gender) - 男/女
- `出生日期` (birth_date) - ISO 8601
- `民族` (ethnicity) - 仅身份证
- `住址` (address) - 仅身份证
- `签发机关` (issuing_authority)
- `有效期` (validity_period)

**特殊能力**:
- ✅ 高精度 OCR (证件专用)
- ✅ 人脸检测 (可选)
- ✅ 印章/钢印检测

**⚠️ 合规提醒**: 身份证号属于敏感个人信息,请遵守《个人信息保护法》,做好脱敏。

**示例响应**:
```json
{
  "scenario": "id_card",
  "fields": {
    "证号": "110101199001011234",
    "姓名": "张三",
    "性别": "男",
    "民族": "汉",
    "出生日期": "1990-01-01",
    "住址": "北京市东城区某某街道123号"
  }
}
```

## 🚚 logistics (物流)

**适用文档**: 快递面单、运单、发货单

**必填字段**:
- `运单号` (tracking_number)
- `收件人` (recipient) - {name, phone, address}
- `寄件人` (sender) - {name, phone, address}

**推荐字段**:
- `品名` (item_description)
- `重量` (weight) - kg
- `运费` (shipping_fee)
- `签收状态` (delivery_status)
- `快递公司` (carrier) - 顺丰/圆通/中通

**特殊能力**:
- ✅ 条形码自动识别 (运单号)
- ✅ 手写体识别 (签名)

## 📄 general_document (通用文档)

**适用文档**: 任意无法明确归类的文档

**必填字段**:
- `标题` (title)

**推荐字段**:
- `摘要` (summary) - LLM 生成的简要概述
- `关键词` (keywords) - 数组
- `作者` (author)
- `创建日期` (created_date)
- `页数` (page_count)
- `语言` (language) - 检测到的语言

**适用场景**:
- 不确定类型时使用
- 自动降级处理
