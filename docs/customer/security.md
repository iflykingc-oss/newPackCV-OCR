# 🔐 隐私与安全

> 客户数据保护是我们的最高优先级。

## 数据保护

### 加密

- **传输加密**: TLS 1.3 (所有 API 通信)
- **存储加密**: AES-256 (静态数据)
- **密钥管理**: HSM (硬件安全模块)
- **密钥轮转**: 90 天自动轮转

### 隔离

- **租户隔离**: 数据库行级安全 (Row-Level Security)
- **存储隔离**: 租户专属 bucket (S3 命名空间)
- **网络隔离**: VPC + 专线可选 (Enterprise)

### 脱敏

- **自动脱敏字段** (默认):
  - 身份证号 → `110101********1234`
  - 手机号 → `138****1234`
  - 银行卡号 → `****1234`
  - 邮箱 → `a***@example.com`
- **自定义脱敏规则**: Enterprise 可配置

## 数据生命周期

```
[上传] → [处理] → [缓存] → [存储] → [删除]
   1min    30s      24h     30天     立即
```

| 阶段 | 时长 | 控制 |
|------|------|------|
| 临时文件 | 处理完立即删除 | 自动 |
| 缓存 | 24 小时 | 自动 |
| 处理结果 | 30 天 (可配置 7/30/90/365) | 客户控制 |
| 审计日志 | 1 年 (Enterprise 可更长) | 不可删除 |
| 备份 | 30 天滚动 | 自动 |

**客户主动删除**: 立即生效,不可恢复 (满足"被遗忘权")。

## 访问控制

### 认证

- API Key (Bearer Token)
- JWT (短期令牌, 24h)
- OAuth 2.0 (第三方应用)
- SAML 2.0 SSO (Enterprise)
- WebAuthn (硬件密钥, Enterprise)

### 授权 (RBAC)

**预置角色**:
- **Owner** - 全部权限
- **Admin** - 除计费/删除外的全部
- **Developer** - API 调用 + 查看日志
- **Analyst** - 只读 (报表/审计)
- **Billing** - 仅计费/发票

**自定义角色** (Enterprise):
- 细粒度权限 (40+ 权限点)
- 字段级权限控制
- IP 白名单

## 审计与可观测

### 审计日志

**记录内容**:
- 何时 (timestamp)
- 谁 (user_id, api_key_id, ip)
- 什么 (action, resource)
- 结果 (success/failure)
- 元数据 (user_agent, request_id)

**不可篡改**: 写入 WORM (Write-Once-Read-Many) 存储

**保留期**:
- Free/Starter: 90 天
- Pro: 1 年
- Enterprise: 自定义 (通常 3-7 年)

### 监控

- **API 调用**: 实时 (Web 控制台)
- **错误率**: 实时告警
- **异常 IP**: 自动告警
- **DDoS**: 自动防护
- **异常请求模式**: AI 检测

### 告警通道

- 邮件
- Webhook
- Slack / 飞书 / 钉钉
- 短信 (Enterprise)

适用场景: 数据合规要求极高,无法上云

### 部署模式

- **VPC 部署**: 您 AWS/阿里云账号内,完全控制
- **专有云**: 独立硬件,我们管理
- **混合云**: 核心数据本地,辅助功能云端

### 包含

- 全部引擎代码
- 运维手册 + 培训
- 1 年技术支持 + 版本更新

### 不包含

- 大型定制: ¥200-500 万
- 年维护费: 15-20% (含版本更新 + 支持)

## 终止与数据迁移

### 主动终止

- 立即停止服务
- 30 天内可下载数据 (CSV/JSON/PDF)
- 30 天后**永久删除**所有数据
- 出具数据删除证明 (Enterprise)

### 数据导出

支持的格式:
- JSON (结构化字段)
- CSV (Excel 可直接打开)
- PDF (原始文档 + 提取结果)
- ZIP (批量打包)

**导出 API**:
```bash
curl -X POST /api/v1/export \
  -H "Authorization: Bearer ..." \
  -d '{"from": "2024-01-01", "to": "2024-01-31", "format": "json"}'
```
